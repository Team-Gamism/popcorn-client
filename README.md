# Popcorn-Pot-Client

게이미즘 프로젝트 **팝콘팟(Popcorn Pot)**의 클라이언트 저장소입니다.

# GitHub 협업 규칙

## 작업 순서

1. 작업할 내용을 GitHub Issue로 등록합니다.
2. `develop` 브랜치를 기준으로 Issue에 맞는 작업 브랜치를 생성합니다.
3. 작업 내용을 커밋하고 원격 저장소에 Push합니다.
4. 작업 브랜치에서 `develop` 브랜치를 대상으로 Pull Request를 생성합니다.
5. 코드 리뷰와 수정 사항 반영을 진행합니다.
6. 리뷰가 완료되면 작업 브랜치를 `develop` 브랜치에 병합합니다.

> 기능 개발 및 수정 작업은 `develop` 브랜치를 기준으로 진행하며, `main` 브랜치에 직접 작업하거나 Pull Request를 생성하지 않습니다.

## 브랜치 규칙

브랜치 이름은 다음 형식을 사용합니다.

```text
타입/작업-내용
```

예시:

```text
feature/player-movement
fix/login-button
refactor/card-manager
chore/unity-settings
```

브랜치 이름은 영문 소문자와 하이픈(`-`) 사용을 권장합니다.

## 커밋 규칙

### 커밋 형식

```text
type: 작업 내용
```

예시:

```text
feat: 플레이어 이동 기능 추가
fix: 시작 버튼이 작동하지 않는 문제 수정
refactor: 게임 매니저 코드 구조 개선
```

### 커밋 타입

| Type       | 사용 상황           |
| ---------- | --------------- |
| `feat`     | 새로운 기능 추가       |
| `fix`      | 버그 수정           |
| `refactor` | 기능 변화 없는 코드 개선  |
| `style`    | 코드 포맷 및 공백 정리   |
| `test`     | 테스트 코드 추가 또는 수정 |
| `chore`    | 빌드, 설정 및 패키지 변경 |

GitHub Desktop을 사용하는 경우 다음과 같이 작성합니다.

* **Summary**: 커밋 타입과 작업 내용을 작성합니다.
* **Description**: 부가 설명과 연관된 Issue 번호를 작성합니다.

예시:

```text
Summary
feat: 카드 선택 기능 추가

Description
#12 카드 클릭 시 선택 상태가 표시되도록 구현
```

추가 설명이 필요하지 않은 경우 Description은 생략할 수 있습니다.

## Pull Request 규칙

### PR 제목

커밋과 동일하게 작업 타입을 포함하여 작성합니다.

```text
[Type] 작업 내용
```

예시:

```text
[Feature] 카드 선택 기능 추가
[Fix] 설정 버튼 클릭 오류 수정
[Refactor] 게임 상태 관리 코드 개선
```

### PR 설정

* **Reviewers**: 작업 내용을 확인할 팀원을 지정합니다.
* **Assignees**: 일반적으로 PR 작성자 본인을 지정합니다.
* **Labels**: 작업 종류에 맞는 라벨을 지정합니다.
### PR 본문 템플릿

> 아래 템플릿은 기본 예시이며, 작업 내용에 따라 항목을 추가하거나 변경하여 사용할 수 있습니다.

```markdown
## 개요

카드 선택 기능을 구현했습니다.

## 작업 내용

- 카드 클릭 입력 처리
- 선택된 카드 UI 표시
- 선택 취소 기능 구현

## 테스트

- 카드 선택 및 선택 취소 정상 작동 확인
- 여러 카드를 연속으로 선택했을 때 상태 확인

## 관련 Issue

- Closes #12
```

## Issue 규칙

Issue 제목에는 작업 종류를 표시합니다.

```text
[Feature] 새로운 기능
[Fix] 버그 수정
[Improve] 기존 기능 개선
[Refactor] 코드 구조 개선
```

### Issue 본문 템플릿

> 아래 템플릿은 기본 예시이며, 작업 내용에 따라 항목을 추가하거나 변경하여 사용할 수 있습니다.

```markdown
## 개요

작업할 기능이나 문제를 간단히 설명합니다.

## 배경

해당 작업이 필요한 이유를 작성합니다.

## TO-DO

- [ ] 첫 번째 작업
- [ ] 두 번째 작업
- [ ] 테스트 및 확인
```

## 작업 시 주의 사항

* `main` 브랜치에서 직접 작업하거나 커밋하지 않습니다.
